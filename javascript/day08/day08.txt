DOM API를 왜 사용하는지

로그인 기능?

최종 구현해야 하는 형태가 뭘까요?
- 버튼을 클릭하면 서버에 아이디와 비밀번호를 전송하는 것
- 이후의 동작도 예상할 수 있어야해요 (보류)

단순히 함수나 가능 단위를 설계하는 것도 중요하지만
서비스 단위에서 어떤 함수들과 어떤 기능이 필요하고, 어떤 예외 상황들이 있는지 예측

버튼 클릭 시 함수가 실행
1. 함수에는 아이디와 비밀번호를 서버에 전송하는 기능
 1-1. 서버에 전송하는 기능
 1-2. 아이디와 비밀번호의 값 -> 입력 요소가

2. 버튼 요소

즉 앞으로 우리는 기능을 구현하는데 필요한 값에 있어서
***특정 요소***에 접근 해야할 필요가 있다

요소에 접근할 수 있는 DOM API가 필요해진 것
